# Skill: Update Flint Init

Update `Mesh/(System) Flint Init.md` to reflect changes made during the current session.

# Input

- Knowledge of what changed during this session (tasks completed, plans updated, increments progressed, new artifacts created, etc.)

# Actions

1. Read the current `Mesh/(System) Flint Init.md`
2. Identify sections that are stale or incomplete based on session activity:
   - Active increment may have changed
   - Active workspaces may have been added or completed
   - Recent milestones may need updating
   - Navigation references may need additions
   - Mandates may have been added, deactivated, or changed — update the `### Mandates` table to list only active mandates with their one-line condition summary
3. Update only the sections that need changes — don't rewrite the entire file
4. Preserve the document's structure and follow [[dev-tmp-f-flint_init]]

# Output

- Updated `Mesh/(System) Flint Init.md` reflecting current session changes
